Unit 01 // Introduction to Computer Networks

Foundations, Types, Topologies & Components

🌐
A computer network is a set of interconnected computing devices that can communicate and share resources using a defined set of rules (protocols). Networks range from two computers linked in a home to billions of devices forming the global internet.

1.1 History & Evolution of Networking

The history of networking spans six decades of remarkable innovation:

  • 1960s: ARPANET — the U.S. Department of Defense funded the first packet-switching network, connecting four universities. This is the direct ancestor of the internet.
  • 1970s: TCP/IP protocol suite developed by Cerf and Kahn; Ethernet invented at Xerox PARC.
  • 1980s: DNS introduced; TCP/IP adopted as the standard; NSFNet expands academic networking.
  • 1990s: Tim Berners-Lee invents the World Wide Web (WWW); commercialization of the internet begins; dial-up modems become widespread.
  • 2000s–present: Broadband, Wi-Fi, mobile internet, cloud computing, IoT, 5G.

1.5 Transmission Media

Wired (Guided) Media:

  • Twisted Pair (UTP/STP): Cat5e supports 1 Gbps up to 100m; Cat6 supports 10 Gbps up to 55m; Cat6a 10 Gbps up to 100m.
  • Coaxial Cable: Used for cable TV and early Ethernet (10BASE2, 10BASE5). Provides better shielding than twisted pair.
  • Fibre Optic: Uses light pulses; single-mode (long distance, ~100km) vs multimode (short distance, up to 2km). Speeds: 10 Gbps to 100 Tbps. Immune to electromagnetic interference.

Wireless (Unguided) Media: Radio waves (Wi-Fi, Bluetooth), Microwave (point-to-point links), Infrared (short range), Satellite (global coverage, high latency ~600ms).

Intro to Networks · 5 Questions
Q1 Multiple Choice
A university campus has multiple buildings connected via a high-speed fibre backbone, all within a 2km radius. Which network type best describes this?
Correct: B — LAN. A campus network spanning a few kilometres is a LAN. A MAN would span an entire city; a WAN spans countries/continents.
Q2 True / False
A network hub operates at Layer 2 (Data Link) of the OSI model and uses MAC addresses to forward frames intelligently.
False. A hub operates at Layer 1 (Physical) and blindly broadcasts all data to every port. It is a switch that operates at Layer 2 using MAC address tables.
Q3 Fill in the Blank
The network topology in which every node has a dedicated connection to every other node, providing maximum redundancy, is called a __________ topology.
Answer: Mesh. In a full mesh topology with n nodes, there are n(n-1)/2 links. It provides maximum fault tolerance but is expensive and complex to manage.
Q4 Multiple Choice
Which transmission medium is completely immune to electromagnetic interference (EMI) and supports the highest bandwidth over the longest distances?
Correct: C. Single-mode fibre uses a laser to transmit light through a very narrow core (~9 microns), enabling distances up to 100km and speeds exceeding 100 Gbps, with zero susceptibility to EMI.
Q5 Short Answer
Compare the Star and Bus topologies in terms of: (a) failure impact, (b) ease of troubleshooting, and (c) scalability. Which is preferred in modern LANs and why?
Model Answer: (a) Failure: In a Bus, a single cable break fails the entire network. In a Star, only the affected node is isolated — the central switch remains operational. (b) Troubleshooting: Star is far easier — you isolate problems port by port at the switch. Bus requires checking the entire cable run. (c) Scalability: Star scales easily — add another port/switch. Bus becomes increasingly inefficient with more nodes. Modern LANs universally use switched Star topology because switches eliminate collisions, provide per-port bandwidth, support VLANs, and enable easy management.

The TCP/IP model (also called the Internet model) was developed pragmatically for ARPANET and collapses the OSI 7 layers into 4 functional layers:

2.4 Data Encapsulation

As data travels down the OSI stack at the sender, each layer adds a header (and sometimes a trailer) to the data — this is called encapsulation. At the receiver, each layer decapsulates (removes its header) as data moves up the stack.

Encapsulation Flow // Application Layer (L7) DATA = "GET /index.html HTTP/1.1" // Transport Layer (L4) adds TCP header SEGMENT = [TCP Header | DATA] // Fields: Source Port, Dest Port, Seq No, Checksum... // Network Layer (L3) adds IP header PACKET = [IP Header | SEGMENT] // Fields: Src IP, Dest IP, TTL, Protocol... // Data Link Layer (L2) adds Ethernet frame FRAME = [Eth Header | PACKET | FCS Trailer] // Fields: Src MAC, Dest MAC, Type | CRC checksum // Physical Layer (L1) BITS = 01001000 01000101 01000001 01000100 ...
Unit 2 Assessment
OSI & TCP/IP Models · 5 Questions
Q1 Multiple Choice
At which OSI layer does a router primarily operate, and what addressing scheme does it use?
Correct: B. Routers operate at Layer 3 (Network layer) and make forwarding decisions based on destination IP addresses in the packet header.
Q2 Fill in the Blank
The process by which each OSI layer adds its own header to the data from the layer above is called __________. The reverse process at the receiver is called __________.
Answer: encapsulation / decapsulation. Encapsulation builds the PDU by wrapping data with headers/trailers at each layer. Decapsulation reverses this, with each layer reading and removing its own header.
Q3 True / False
The TCP/IP model's "Network Access" layer combines the functions of BOTH OSI Layer 1 (Physical) and Layer 2 (Data Link).
True. The TCP/IP Network Access layer (also called Link layer) handles both physical transmission (OSI L1) and frame-level addressing/error detection (OSI L2).
Q4 Multiple Choice
SSL/TLS encryption operates at which OSI layer?
Correct: C. The Presentation layer handles data format translation, encryption/decryption, and compression. SSL/TLS are Presentation layer protocols that provide encryption services to Application layer protocols like HTTP.
Q5 Short Answer
Explain why the OSI model has 7 layers but the TCP/IP model has only 4. What happened to OSI layers 5 and 6 in TCP/IP?
Model Answer: The OSI model is a theoretical reference framework designed for conceptual clarity — each layer has a precise, distinct function. The TCP/IP model was developed pragmatically for ARPANET and reflects how internet protocols actually work in practice. OSI layers 5 (Session) and 6 (Presentation) were collapsed into TCP/IP's Application layer because in practice, most internet applications handle session management and data formatting themselves (e.g., HTTP manages its own session; TLS handles encryption at the application level). The TCP/IP model is not inferior — it is simply a more pragmatic, implementation-focused abstraction.

Unit 03 // Physical & Data Link Layer

Signals, Encoding, Error Detection, MAC & Ethernet

⚡
The Physical layer transmits raw bits over a medium. The Data Link layer packages bits into frames, handles physical addressing (MAC), and provides error detection/correction between directly connected nodes.

3.1 Analog vs Digital Signals

An analog signal is continuous, varying smoothly over time. A digital signal is discrete, representing data as binary values (0s and 1s). Modern networks use digital signals but must convert them for transmission over analog media (e.g., telephone lines) using modulation.

📊 Fig 3.1 — Analog (continuous) vs Digital (discrete) signal waveforms (Wikimedia Commons)

3.2 Bandwidth, Latency & Throughput

  • Bandwidth: Maximum data transfer rate of a medium — measured in bits per second (bps, Mbps, Gbps). This is the theoretical maximum capacity.
  • Throughput: Actual data transfer rate achieved in practice — always less than bandwidth due to overhead, errors, congestion.
  • Latency: The time delay for a packet to travel from source to destination — measured in milliseconds (ms).
  • Goodput: Useful data throughput excluding retransmissions and protocol overhead.
Shannon's Channel Capacity Formula // Maximum data rate (bits/sec) for a noisy channel: C = B × log₂(1 + S/N) Where: C = channel capacity (bps) B = bandwidth of channel (Hz) S/N = Signal-to-Noise ratio // Example: B=3000Hz, SNR=30dB (S/N = 1000) C = 3000 × log₂(1 + 1000) C = 3000 × 9.97 C ≈ 29,900 bps ≈ 30 Kbps

3.4 MAC Addressing

A MAC (Media Access Control) address is a 48-bit (6-byte) hardware address permanently assigned to a network interface card (NIC). It is expressed as 6 pairs of hexadecimal digits.

MAC Address Structure // Format: XX:XX:XX:XX:XX:XX (colon notation) // Or: XX-XX-XX-XX-XX-XX (hyphen notation) Example: 00:1A:2B:3C:4D:5E └──────────┘ └──────────┘ OUI (Org ID) Device ID (3 bytes) (3 bytes) // OUI = Organizationally Unique Identifier // Assigned by IEEE to manufacturers 00:1A:2B → Apple Inc. 00:50:56 → VMware FC:FB:FB → Cisco Systems // Special addresses: FF:FF:FF:FF:FF:FF → Broadcast (all devices) 01:00:00:00:00:00 → Multicast (prefix)

3.5 Ethernet & IEEE 802.3

Ethernet is the dominant LAN technology, standardized as IEEE 802.3. Modern Ethernet uses CSMA/CD (Carrier Sense Multiple Access with Collision Detection) though full-duplex switched Ethernet eliminates collisions entirely.

📦 Fig 3.2 — Ethernet Type II Frame Format showing all fields (Wikimedia Commons)

3.6 VLANs

A VLAN (Virtual LAN) logically segments a physical network into multiple isolated broadcast domains. VLANs improve security, reduce broadcast traffic, and enable flexible network design without physical rewiring. IEEE 802.1Q adds a 4-byte VLAN tag to Ethernet frames: 12 bits for VLAN ID (supporting VLANs 1–4094).

Unit 3 Assessment
Physical & Data Link · 5 Questions
Q1 Multiple Choice
A channel has a bandwidth of 4000 Hz and a Signal-to-Noise ratio of 255. Using Shannon's theorem, what is the maximum channel capacity?
Correct: C — 32,000 bps. C = B × log₂(1 + S/N) = 4000 × log₂(256) = 4000 × 8 = 32,000 bps. Note: log₂(256) = 8 because 2⁸ = 256.
Q2 Fill in the Blank
In a MAC address, the first 3 bytes (24 bits) identify the manufacturer and are called the __________ (OUI).
Answer: Organizationally Unique Identifier. The OUI is the first 24 bits of a MAC address, assigned by IEEE to hardware manufacturers. The last 24 bits are assigned by the manufacturer to uniquely identify the device.
Q3 True / False
CRC (Cyclic Redundancy Check) can both detect AND automatically correct burst errors in transmitted data.
False. CRC is an error detection scheme only — it can detect burst errors up to r bits (where r is the degree of the generator polynomial) but cannot correct them. When an error is detected, the frame is discarded and retransmission is requested.
Q4 Multiple Choice
How many VLANs can be created using the 12-bit VLAN ID field in the IEEE 802.1Q standard?
Correct: C — 4094. A 12-bit field gives 2¹² = 4096 possible values (0–4095). VLANs 0 and 4095 are reserved, leaving 4094 usable VLANs (1–4094).
Q5 Short Answer
Explain the difference between bandwidth and throughput. Give a practical analogy and explain why throughput is always less than bandwidth in real networks.
Model Answer: Bandwidth is the theoretical maximum capacity of a network link (e.g., 1 Gbps). Throughput is the actual achieved data transfer rate (e.g., 750 Mbps). Analogy: A highway with 6 lanes (bandwidth) can theoretically accommodate maximum traffic, but actual traffic flow (throughput) is reduced by toll booths (protocol overhead), accidents (packet loss/retransmissions), speed limits (congestion control), and merging slowdowns (queuing delays). Throughput is always less than bandwidth due to: protocol headers consuming bandwidth; packet retransmissions due to errors; congestion control throttling; queuing and processing delays at intermediate nodes; and TCP acknowledgment overhead.

Unit 04 // Network Layer

IP Addressing, Subnetting, CIDR, VLSM & Routing

🗺️
The Network Layer (OSI Layer 3) provides logical addressing, routing, and packet forwarding between networks. The Internet Protocol (IP) is the foundation — every device on an IP network has a unique IP address used to identify it and route packets to it.

4.1 IPv4 Addressing

An IPv4 address is a 32-bit number represented in dotted-decimal notation — four 8-bit octets separated by dots.

IPv4 Address Structure // 32-bit binary representation: IP: 192.168.10.5 Binary: 11000000.10101000.00001010.00000101 Octet 1: 192 = 1+1+0+0+0+0+0+0 = 11000000 Octet 2: 168 = 10101000 Octet 3: 10 = 00001010 Octet 4: 5 = 00000101 // IPv4 Address Classes (Classful): Class A: 1.0.0.0 – 126.255.255.255 /8 (16M hosts) Class B: 128.0.0.0 – 191.255.255.255 /16 (65K hosts) Class C: 192.0.0.0 – 223.255.255.255 /24 (254 hosts) Class D: 224.0.0.0 – 239.255.255.255 Multicast Class E: 240.0.0.0 – 255.255.255.255 Reserved

4.2 Subnet Masks & CIDR Notation

A subnet mask defines which portion of an IP address is the network portion and which is the host portion. CIDR (Classless Inter-Domain Routing) notation expresses the mask as a prefix length (e.g., /24).

🔢 Subnetting Calculation — Step by Step
Problem: Network 192.168.1.0/24 — divide into 4 equal subnets.
Step 1: Need 4 subnets → 2² = 4 → borrow 2 bits from host portion
Step 2: New prefix = /24 + 2 = /26
Step 3: New subnet mask = 255.255.255.192 (11000000 in last octet)
Step 4: Hosts per subnet = 2^(32-26) - 2 = 2^6 - 2 = 64 - 2 = 62 usable hosts
Step 5: Subnet block size = 256 - 192 = 64
Subnet 1: 192.168.1.0/26 → Hosts: .1–.62 → Broadcast: .63
Subnet 2: 192.168.1.64/26 → Hosts: .65–.126 → Broadcast: .127
Subnet 3: 192.168.1.128/26 → Hosts: .129–.190 → Broadcast: .191
Subnet 4: 192.168.1.192/26 → Hosts: .193–.254 → Broadcast: .255

4.3 VLSM — Variable Length Subnet Masking

VLSM allows a network to use different subnet mask lengths within the same network — allocating exactly the right number of host addresses for each subnet, minimizing waste. This is the advanced enterprise standard.

🔢 VLSM Calculation Example
Given: 192.168.1.0/24 | Requirements:
• Network A: 100 hosts | • Network B: 50 hosts
• Network C: 25 hosts | • Network D: 10 hosts (2× WAN links)
──────────────────────────────────────
Sort LARGEST first (VLSM rule):
Network A (100 hosts): Need 2^n-2 ≥ 100 → 2^7=128 → /25
→ 192.168.1.0/25 Hosts: .1–.126 BC: .127 (126 hosts)
Network B (50 hosts): Need 2^n-2 ≥ 50 → 2^6=64 → /26
→ 192.168.1.128/26 Hosts: .129–.190 BC: .191 (62 hosts)
Network C (25 hosts): Need 2^n-2 ≥ 25 → 2^5=32 → /27
→ 192.168.1.192/27 Hosts: .193–.222 BC: .223 (30 hosts)
Network D (10 hosts): Need 2^n-2 ≥ 10 → 2^4=16 → /28
→ 192.168.1.224/28 Hosts: .225–.238 BC: .239 (14 hosts)
WAN links (2 hosts each): 2^2-2=2 → /30
→ 192.168.1.240/30 + 192.168.1.244/30

4.4 Wildcard Masks

A wildcard mask is the inverse of a subnet mask — 0 bits mean "must match" and 1 bits mean "any value." Wildcard masks are used in ACLs (Access Control Lists) and OSPF configurations.

Wildcard Mask Calculation // Wildcard = 255.255.255.255 - Subnet Mask Subnet mask /24: 255.255.255.0 Wildcard: 0.0.0.255 // Matches any host in the /24 network Subnet mask /26: 255.255.255.192 Wildcard: 0.0.0.63 // Matches hosts .0 to .63 in the subnet Subnet mask /30: 255.255.255.252 Wildcard: 0.0.0.3 // Matches only 4 addresses (WAN link) // ACL example using wildcard: access-list 10 permit 192.168.1.0 0.0.0.255 // Permits all hosts in 192.168.1.0/24

4.5 IPv6 Addressing

IPv6 uses 128-bit addresses represented as 8 groups of 4 hexadecimal digits. The address space is 2¹²⁸ ≈ 340 undecillion addresses — effectively unlimited.

IPv6 Address Format Full: 2001:0db8:85a3:0000:0000:8a2e:0370:7334 Compressed: 2001:db8:85a3::8a2e:370:7334 // Rules: Remove leading zeros; :: replaces ONE sequence of all-zero groups Loopback: ::1 (equivalent to 127.0.0.1 in IPv4) Link-local: fe80::/10 (auto-configured, not routable) Global unicast: 2000::/3 // Prefix notation same as IPv4 CIDR: 2001:db8::/32 // Documentation prefix

Q1 Calculation — CIDR
Given the network 172.16.0.0/20, calculate: (a) the subnet mask in dotted-decimal, (b) the number of usable host addresses, (c) the broadcast address.
Solution: (a) /20 = 20 bits network, 12 bits host. First 20 bits set = 11111111.11111111.11110000.00000000 = 255.255.240.0. (b) Usable hosts = 2^12 - 2 = 4096 - 2 = 4,094 hosts. (c) Network: 172.16.0.0. Last host octet block: 256-240=16. Broadcast = 172.16.0.0 + 16 addresses in 3rd octet - 1 = 172.16.15.255.
Q2 Calculation — VLSM
You are given 10.0.0.0/24. Using VLSM, allocate subnets for: Dept A = 60 hosts, Dept B = 28 hosts, Dept C = 12 hosts, WAN link = 2 hosts. Show the subnet address, mask, and usable range for each.
Solution (largest first): Dept A (60 hosts): 2^6=64 → /26. 10.0.0.0/26, mask 255.255.255.192, hosts 10.0.0.1–.62, BC .63. | Dept B (28 hosts): 2^5=32 → /27. 10.0.0.64/27, mask 255.255.255.224, hosts .65–.94, BC .95. | Dept C (12 hosts): 2^4=16 → /28. 10.0.0.96/28, mask 255.255.255.240, hosts .97–.110, BC .111. | WAN (2 hosts): 2^2=4 → /30. 10.0.0.112/30, mask 255.255.255.252, hosts .113–.114, BC .115.
Q3 Multiple Choice
What is the wildcard mask for the subnet 192.168.5.128/27?
Correct: C — 0.0.0.31. /27 subnet mask = 255.255.255.224. Wildcard = 255.255.255.255 - 255.255.255.224 = 0.0.0.31. In binary: 224 = 11100000, so wildcard last octet = 00011111 = 31.
Q4 True / False
BGP (Border Gateway Protocol) is classified as an Interior Gateway Protocol (IGP) used within a single autonomous system.
False. BGP is an Exterior Gateway Protocol (EGP) — specifically a path-vector protocol used to route between autonomous systems (ASes). It is the routing protocol of the internet backbone.
Q5 Fill in the Blank
IPv6 uses ________-bit addresses, providing approximately 3.4 × 10^38 unique addresses, solving IPv4 exhaustion.
Answer: 128. IPv6 addresses are 128 bits, compared to IPv4's 32 bits. This provides 2^128 ≈ 3.4 × 10^38 addresses — enough for every atom on Earth's surface to have millions of addresses.
Q6 Binary Conversion
Convert the IPv4 address 10.255.128.5 to binary. Then identify which class it belongs to and its default subnet mask.
Solution: 10 = 00001010 | 255 = 11111111 | 128 = 10000000 | 5 = 00000101. Binary: 00001010.11111111.10000000.00000101. The first octet is 10 (00001010), which falls in the range 1–126 → Class A. Default Class A mask = 255.0.0.0 (/8). Note: 10.x.x.x is also a private (RFC 1918) address range.
